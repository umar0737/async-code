## Синхрони код ин кодҳое мебошад ки бе ожидани кор мекунад яъне аввал лайни якум кор мекунад баъд лайнҳо дигар. 
## Асинхрони кодҳо кодҳое мебошанд ки ожидани дора
![alt](https://enlightsolution.com/wp-content/uploads/2019/05/Synchronous-vs.-asynchronous.jpg)
## Мо се роҳи коди асинхрони сохтанро дорем
## Якум роҳ ин бо воситаи callback. Бо воситаи калбэк ин вобаста бa функсия кор мекунад. Яъне ҳамаи функсияе ки калбэк дошта бошад ин маъно надорад ки вай асинхрони аст.
![alt](https://media.geeksforgeeks.org/wp-content/uploads/20210328211825/async1.png)
## Дуюм роҳ ин бо воситаи Promise мебошад. Сохтани promise ин бо воситаи сохтани конструктор мебошад. Промис се состояния дорад: Pending, fulfiled(resolve) ва rejеcted. Масалан вақте ки мо промис сохта логикаи худро менависем у panding ба мо дар ҷавоб fulfiled(resolve) ё rejected бар мегардонад.

## Сеюм роҳ ин бо воситаи функсияи async/await мебошад. async ин як keyword мебошад ки функсияи моро асинхрони мекунад. 
![alt](https://cdn-images-1.medium.com/max/1026/1*-JQKo_gKJvErqJBOeI9JcA.png)

## Try catch - ин як конструктор аст ки дар он мо коди худро менависем. Агар дар блоки try ягон ошибка шава блоки catch кор мекунад.

## api/rest API - api ин як механизм мебошад ки байни фронтенд ва бэкенд ялоқа пайваст мекунад.
## rest - ин як набор қоидаҳое мебошад ки барои сохтани алоқа байни истифодабаранда ва сервер лозим аст.
## fetch - ин як методи асинхрони код мебошад ки ба api - доступ медиҳад.
